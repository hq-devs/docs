## §3 P1 · 启动 / 欢迎 / 工程列表

### §3.1 冷启动落地页（首次无配置）

判定方式：检查 `localStorage`（键 `ptw.config`）是否存在 `onboarded` 标记；正式版由 Go 侧 `GetConfig` 读取。

**不强弹向导**：落地页常驻显示，待用户主动点击「新建工程」才触发向导。

| 区块 | 内容 |
|---|---|
| 欢迎标题 | 「欢迎使用 · 点表智能工作台」+ 价值一句话（把协议文档交给 AI、连设备逐条对答案）|
| 双入口大卡 | 「新建工程」（虚线主卡，点击触发向导）+ 「打开工程」（已有本地目录直接打开）|
| 上手指引卡 | 「现场四步就收工」（见 §3.4）|
| 平台连接 chip | 冷启动默认「离线（未连接平台）」；预填示例地址，需主动「测试连接」才算已连 |

### §3.2 新建工程向导（按需弹出，可随时关闭）

向导为浮层模态，X 按钮与点遮罩均可取消，不绑定 onboarded 状态（用户可随时关闭）。

**步骤①：工程目录（必填）**

| 字段 | 说明 |
|---|---|
| 工程名 | 文本输入，必填 |
| 甲方名称 | 文本输入，可选 |
| 工程目录 | 「选择文件夹」按钮（正式版调 Go 侧 `SelectProjectDir`）；未选目录禁用「下一步」|
| 说明文案 | 本地工程目录：点表产物（JSON DSL/xlsx）、会话与日志都落在此目录；AI 建表与真机调试由云端完成 |
| 黄色提示条（条件显示） | 若未连平台时点开向导，步骤①顶部出现：「AI 解析建表与真机调试都在公司云端（本机免装大模型），需连接才能用；本地仅查看/编辑已有点表。可在下一步连接，或随时点右上角 chip」；可一键关闭，不阻断 |

**步骤②：平台连接（云端能力入口）**

| 字段 | 说明 |
|---|---|
| 服务器地址 | 文本输入（默认空，非预填）|
| projectKey | 文本输入 |
| 账号 | 文本输入 |
| 「测试连接」按钮 | 连接成功显示绿色徽章 |
| 「稍后连接」跳过链接 | 文案：「稍后连接（仅查看已有点表）→」；点击完成向导但不配置远端 |
| 信息条 | `cloud-cog` 图标 + 「平台 = 云端大脑（AI 建表/真机调试/提交，本机零配置）」|

**步骤条**：顶部「工程目录 ① → 平台连接 ②」进度指示。

完成后写 `localStorage`（`ptw.config`：`onboarded/projectDir/remote`）并跳 `overview.html`（P2 工程总览）。

## §4 P2 · 工程总览

### §4.1 布局

| 区块 | 内容 |
|---|---|
| 工程信息头 | 工程名、甲方名称、本地路径、创建时间；整体进度条（按任务状态加权）|
| 统计卡片行 | 任务总数、各状态计数（点击筛选下方列表）、工程用量（如 `¥86`）、累计调试时长 |
| 协议点表任务列表 | 主体表格（见 §4.2）；每一行代表一次协议导入形成的独立点表任务 |
| 工具栏 | 「导入文档建点表」（创建新任务并自动生成）/ 搜索框 / 状态筛选 / 批量导出 |
| 顶部连接 chip | 同 §2.4，P2 头部也有一份平台连接 chip |

**对象边界**：

- P2 工程总览只聚合一个工程下的多个协议点表任务，不承载具体点表编辑和调试。
- 点击「导入文档建点表」后，系统在当前工程下创建一个新的 `task_id`，上传协议、解析完成并自动触发首次生成。
- 表格中的每一行都是一个独立任务；它有自己的协议文档、生成 run、JSON DSL、调试会话、确认记录和提交状态。
- P3 单点表工作台始终绑定一行任务，所有编辑、调试、验收动作只影响该任务。

### §4.2 协议点表任务列表列定义

| 列 | 内容 | 说明 |
|---|---|---|
| 任务/设备名称 | 厂家 + 型号 + 任务名 | 一次协议导入形成一个任务，通常对应一个设备/型号的点表 |
| 厂家 / 型号 | — | 可排序 |
| 协议类型 | modbusRTU / modbusTCP 等 | — |
| 七级分类码 | 如 `2.4.4.7.1.6.2` | — |
| 状态徽章 | 7 态之一（见 §2 状态机）| 带颜色徽章 |
| 点位数 | 读 N / 写 N | — |
| 校验错误数 | 协议采数 X + 测点描述 Y | 分两色显示 |
| 最后更新时间 | 相对时间（如「10 分钟前」）| — |
| 操作 | 打开 / 导出 JSON / 导出 xlsx / 删除 | 悬浮显示 |

**交互**：双击行进入该任务的 P3 单点表工作台，默认落在该任务当前状态对应的视图。删除任务需二次确认并提示本地文件去向。

## §5 P3 · 单点表工作台（核心，详细描述）

### §5.1 顶部状态条

```
←返回  [设备名][状态徽章]  |  AI进度分段条  |  读N·写N 通过N 可疑N 失败N 未采N  |  [待澄清N▾]  |  [提交至host▾]  导出  确认提交
```

**AI 进度分段条**：

- 紧凑单行：`✦ ▬▬▬▬▬ AI 3/5 · 调试验证中 ▾`
- 五段细色块按阶段着色：完成=绿 / 进行中=蓝色脉冲 / 待处理=琥珀 / 待开始=灰
- 有告警时图标与文案转琥珀色
- 点击展开弹层：竖排 5 个阶段（序号+名称+状态徽章+一句话说明）

**5 个阶段**：文档解析 → 点位提取 → 字段补全 → 规则校验 → 调试验证

**[待澄清N▾] chip**：

- 状态：`animate-pulse` 脉冲动画（存在未决澄清时）
- 点击：重新打开澄清遮罩门

**[提交至host▾] chip**：

- 回显：绿点=已连接 + 主机名 + ▾
- 点击：展开弹层（就地配置服务器地址/projectKey/账号 + 测试连接 + 连接态）
- 与「导出/确认提交」同处顶部操作区

**确认提交按钮**：

- 门禁联动：可疑/失败/未采 > 0 或未处理澄清项时 disabled，悬浮提示原因
- 「N 项待澄清未决」明示在按钮旁

### §5.2 澄清条（展开形态）

位于顶部状态条下方，展开后插入澄清遮罩门（`phase=confirm`）：

- **遮罩门**：`fixed` 覆盖 body（`top-12` 让出顶栏），三栏 `opacity-50 pointer-events-none`
- **卡片结构**：问题 + 证据（点击跳全宽对照预览高亮）+ **AI 推荐项蓝色高亮 + 一句理由** + 影响点位名列表
- **操作**：逐条选候选答案 / 顶部「全部采纳推荐」（全清后直接进工作台）
- **进度条**：已澄清 x / 总数 y；按类型分布标签（型号适用性/功能码/数据类型/字节序/变比/分类/其他）
- **硬阻塞**：`clarifyCount > 0` 时「进入工作台」按钮禁用

### §5.3 左栏（240px）

**设备元信息区块**：

| 字段 | 说明 |
|---|---|
| 设备名称 | 显示，可编辑 |
| 厂家 / 型号 | 显示 |
| 协议类型 | 显示 |
| 七级分类码 | 显示；点击跳 D4 分类选择器 |
| 状态徽章 | 任务状态（唯一权威状态出口）|
| 「查看完整 ›」链接 | 切中栏到「设备信息」Sheet |

**协议文档区块**（T1a 形态）：

| 元素 | 说明 |
|---|---|
| 文档列表 | 每行：解析状态点（绿=已解析/黄闪=解析中/红=失败）/ 文件名 / 角色标签（主文档/补充/变更说明/抓包记录）/ 页数 / 大小 |
| 点击已解析文档 | 打开全宽对照预览抽屉（覆盖中+右栏）|
| 「+ 上传文档」 | 按钮 + 拖拽区；上传即进入 OCR 解析 |
| 语义说明文案 | 「补充文档不替换原文档：上传后 AI 仅分析增量影响，逐条确认后生效」|

**协议参数区块**：

## §8 AI 留痕规范

### §8.1 假设-证据-结果三元组（固定格式）

每次 AI 填写或修改字段，都必须产生一个三元组记录：

```
假设：[一句话描述 AI 的推断]
证据：[文档名] 第 [N] 页 [标题/表格名]：「[相关原文摘录]」
结果：[字段名] [旧值] → [新值]
```

示例：

```
假设：A相电流变比应为 0.01（寄存器值×0.01得真实电流，单位A）
证据：艾默生060KVA列头柜协议V2.3.pdf 第12页 §5.1模拟量寄存器定义：「注②：电流分辨率 0.01A（即寄存器值 ×0.01）」
结果：变比(乘法) 0.1 → 0.01
```

### §8.2 三元组的存储与显示位置

| 位置 | 说明 |
|---|---|
| 中栏：点位行内 AI 角标 | 显示判定结论（通过/可疑/失败），悬浮可看一句话原因 |
| 右栏：修正建议卡 | 完整三元组 + diff 表 + 操作按钮 |
| 右侧：证据面板 | 选中单元格时显示该字段的完整 AI 填写轨迹（历史三元组列表）|
| 澄清条：每条澄清项 | 「证据」子区块，点击跳转对照预览高亮 |
| 变更影响清单：每条项 | 证据跳转链接 |

### §8.3 证据跳转规范

所有证据链接跳转到**协议文档对照预览抽屉**，并定位到「文档+页码」：

1. 切换到对应文档（如果多个文档）
2. 跳转到对应页码
3. 相关片段黄色高亮（原文侧）
4. 对应 OCR 还原文本琥珀色高亮（OCR 侧）

证据引用格式（用于跳转 URL 或内部引用）：`{docId}:p{pageNo}:{fragmentId}`
